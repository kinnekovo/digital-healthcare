🛠️ 开发者说明 / FastAPI 集成指南 
架构概览（录音流 ASR：Web Speech API + 简洁失败重试 + Web Speech Synthesis TTS）
=========================================================================

训练流程（train.js）                 →  真实 FastAPI 端点（替换位置）
─────────────────────────────────────────────────────────────────────────
doScoring() — state.asrText         →  POST /api/asr
                                           body: FormData { audio, expected_keywords }
                                           resp: { text: string, confidence: number }
                                           (替换 Web Speech API 路径；
                                            写入 state.asrText / state.asrConfidence
                                            后同样触发 doScoring)

MOCK_TTS(text)                       →  POST /api/tts
                                           body: { text: string }
                                           resp: { audio_url: string }
                                           (当前同时使用 Web Speech Synthesis 朗读，
                                            tts.js 封装 speechSynthesis，
                                            设置页可关闭/调速)

MOCK_SCORE(text, conf, kw, dur)      →  POST /api/score
                                           body: { asr_text, confidence,
                                                   expected_keywords, duration_ms }
                                           resp: { score, label, feedback, tip }

ASR 说明（方案1 — 录音流）：
  桌面 Chrome 使用 SpeechRecognition（lang: zh-CN, continuous: false）。
  用户手势触发"开始录音" → recognition.start()。
  "停止录音" → recognition.stop()；显示"识别中…"；onend 处理结果。
  识别成功 → 显示文字，启用"确认并评分"。
  失败/空结果 → 显示错误信息 + "再试一次"大按钮。
  不支持时自动启用"确认并评分"（空文本评分）。

每轮保存字段：asr_text, asr_source (speech|none), asr_error。

统一返回格式：
{ "code": 200, "msg": "ok", "data": { ... } }

场景脚本：
  编辑 scenes.json 即可增删改场景和对话轮次，无需修改 JS。
  字段：id, name, icon, turns[].robot_text / hint / keywords

本地运行：
  python3 -m http.server 8080
  # 访问 http://localhost:8080/
