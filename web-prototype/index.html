🛠️ 开发者说明 / FastAPI 集成指南 
架构概览（当前为纯前端 Mock 实现）
======================================

当前 Mock 函数（app.js）         →  真实 FastAPI 端点（替换位置）
─────────────────────────────────────────────────────────────────
MOCK_TTS(text)                   →  POST /api/tts
                                       body: { text: string }
                                       resp: { audio_url: string }

MOCK_ASR(audioBlob, keywords)    →  POST /api/asr
                                       body: FormData { audio, expected_keywords }
                                       resp: { text: string, confidence: number }

MOCK_SCORE(text, conf, kw, dur)  →  POST /api/score
                                       body: { asr_text, confidence,
                                               expected_keywords, duration_ms }
                                       resp: { score, label, feedback, tip }

统一返回格式：
{ "code": 200, "msg": "ok", "data": { ... } }

替换步骤：
1. 在 app.js 中找到三个 /* API_HOOK */ 注释的函数。
2. 将函数体替换为 fetch('/api/...', {...}) 调用。
3. 处理错误码（401 未授权 / 429 限流 / 500 服务异常）。
4. 在后端：使用 FastAPI + Pydantic 定义请求/响应模型。

口型同步原理：
─────────────
1. HTMLAudioElement 连接到 Web Audio API AnalyserNode。
2. 每帧读取 TimeDomain 数据 → 计算 RMS 均方根振幅。
3. 低通滤波（指数平滑系数 0.25）消除抖动。
4. 将 0..0.3 RMS 映射到 0..1 张嘴比例。
5. 设置 SVG #mouth-open ellipse 的 ry 属性（0..14px）。

如需更高精度口型同步：
• TTS 端点返回 viseme_timeline: [{time_ms, viseme_id}, ...]
• 前端按时间轴切换 SVG 口型状态（需在 SVG 中添加多组口型图层）

本地运行：
─────────
  python3 -m http.server 8080
  # 访问 http://localhost:8080/web-prototype/

浏览器要求：
  Chrome 66+ / Firefox 60+ / Safari 14.1+ / Edge 79+
  需要 HTTPS 或 localhost 才能访问麦克风权限。

替换 Avatar（虚拟形象）：
  1. 准备 200×260 SVG，在口部添加 id="mouth-open" 的 ellipse 元素。
  2. 替换 assets/avatar.svg 文件。
  3. ry 属性会由 JS 动态设置（0..14）。

替换示例音频：
  1. 将 WAV/MP3 文件放入 assets/ 目录。
  2. 修改 app.js 顶部的 SAMPLE_AUDIO_URL 常量。
  3. 也可在 app.js 的 MOCK_TTS() 中指定不同音频 URL。

数据存储：
  所有训练记录存储在浏览器 localStorage (key: rehab_sessions_v1)。
  格式：Session[] → Turn[] 嵌套结构。
  接入后端后可改为 POST /api/session/save 同步到数据库。
